настройка сурса и таргет для голденгейта:
------------------------------------------------
SOURCE
-- 1. Enable GoldenGate Replication
  alter system set enable_goldengate_replication=true scope=both;
  
  -- 2. Force Logging for data safety
  alter database force logging;
--ПОДКЛЮЧИТЬСЯ К СДБ, ПОТОМ ОТКЛЮЧИТЬСЯ И СНОВА ПДБ
  alter database add supplemental log data;

  -- 3. Create GoldenGate User
  create user ggadmin identified by Kate2006;
  grant dba to ggadmin;
  grant connect, resource to ggadmin;
  
  -- 4. Prepare for Schema Replication (Example Schema: HR)
  create user hr identified by hr;
  grant connect, resource to hr;
  alter user hr quota unlimited on users;
  create table hr.employees (id number primary key, name varchar2(50));
  exit;

TARGET
-- 1. Enable GoldenGate Replication
  alter system set enable_goldengate_replication=true scope=both;

  -- 2. Create GoldenGate User
  create user ggadmin identified by Kate2006;
  grant dba to ggadmin;
  grant connect, resource to ggadmin;

  -- 3. Create Target Schema (Must exist to receive data)
  create user hr identified by hr;
  grant connect, resource to hr;
  alter user hr quota unlimited on users;
  create table hr.employees (id number primary key, name varchar2(50));
  exit;

в таблице, которая была создана, лежит значение имени. оно НЕ реплицируется, тк репликация проходит только после своего начала. То есть, то, что было добавлено до ее начала, добавлено в другую субд не будет!!!!!
